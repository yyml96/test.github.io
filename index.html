  1. What are the obligatory elements in each (X)HTML document?

    2. Each XHTML document must have the following elements:

    • A doctype declaration indicating that the document is written in XHTML.
    • The html element, which serves as the root element of the document.
    • The head element, which contains metadata about the document.
    • The title element, which provides a title for the document that is displayed in the browser's title bar or tab.
    • The body element, which contains the content of the document that is displayed in the browser window. It is also good practice to include the xmlns attribute on the html element,to specify the XML namespace for the document.

  2. What's DOCTYPE declaration meant for?

    3. The DOCTYPE declaration is an instruction to the web browser about what version of HTML or XHTML the document is written in. It is used to ensure that the browser renders the content correctly, by specifying the Document Type Definition (DTD) that the document conforms to. The DTD defines the rules for the markup language, such as what elements and attributes are available, and how they can be used. The browser uses the DTD to validate the document and ensure that it is well-formed and follows the correct syntax.

  3. What is an URL? Give examples both of use of absolute and relative URLs.

    4. A URL, or uniform resource locator, is a string of text that identifies the location of a resource on the internet, such as a webpage, image, or file. A URL typically includes the protocol (such as http or https), the domain name, and the path to the resource.

    An Absolute URL

    An absolute URL includes the full address of the resource, including the protocol and domain name. For example: https://www.lab.fi/en/info This URL specifies that the resource can be accessed using the HTTPS protocol and the domain name www.lab.fi, and that the resource is located in the "en/info" directory.

    A Relative URL

    A relative URL, on the other hand, does not include the protocol and domain name and instead, it's based on the location of the current page. For example: "en/info" This URL specifies that the resource is located in the "en/info" directory, relative to the current page. The browser will use the protocol and domain name of the current page to complete the URL.Relative URLs can be useful when linking to resources within the same website, as the links will still work even if the domain name changes. Absolute URLs are useful when linking to external resources, as the links will not break if the domain name changes.

  7. There are three different ways of adding CSS styles on an (X)HTML page. How? What are the benefits of each way?

    1. Inline styles

      2. Inline styles are added directly to the HTML elements using the style attribute. They are useful when you need to apply a specific style to a single element and you don't want to create a separate CSS class for it.

      For example:

      <p style="color: blue;">This is a blue paragraph.</p>

    2. Internal stylesheet

      3. An internal stylesheet is added to the <head> section of the HTML document using the <style> element. It is useful when you want to apply styles to multiple elements on the same HTML page and you don't need to reuse the styles on other pages.

      For Example:

      <head> <style> p { color: blue; } </style> </head>

    3. External stylesheet

      4. An external stylesheet is a separate CSS file that is linked to the HTML document using the <link> element in the <head> section of the HTML document. External stylesheets are useful when you want to apply styles to multiple pages or when you want to separate the presentation from the structure of the HTML document.

      For Example:

      <head> <link rel="stylesheet" href="styles.css"> </head
